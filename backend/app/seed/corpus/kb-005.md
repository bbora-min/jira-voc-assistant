## 결론 (요약)
- **외부 공급망 / 정부·금융 클라이언트 제출용** : SPDX 2.3 권장 (NIST·EU 권고 포맷)
- **내부 분석 / DevSecOps 파이프라인 통합** : CycloneDX 1.5 권장 (취약점 메타데이터 풍부)

## 비교 표
| 항목 | SPDX 2.3 | CycloneDX 1.5 |
|---|---|---|
| 표준화 | ISO/IEC 5962 | OWASP 표준 |
| 라이선스 표현 | 강함 (SPDX expression) | 기본 |
| 취약점(VEX) 통합 | 별도 문서 | inline 지원 |
| JSON · XML | 둘 다 지원 | 둘 다 지원 |

## 권장 워크플로
1. 1차 출력은 CycloneDX로 생성 (내부 분석/취약점 관리)
2. 외부 제출 시 `sbom-cli convert --to spdx-2.3` 로 변환
3. 클라이언트 요구 포맷이 명시되어 있다면 그 포맷을 우선 따른다
